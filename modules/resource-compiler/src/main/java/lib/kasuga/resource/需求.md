# ResourceCompiler 需求
## 准备阶段
在拿到所有的顶层 modules.yml 之后:
1. 首先使用 TheElectronWill/night-config 读取 toml 文件，目前无配置，以后可能会有
2. 对所有 module 使用 ForkJoinPool 扫描，构造一个 record SourceResource(ResourceCategory category, Path relativePath, File file), 其中:
某文件如果是 blockstates/mkz_bogey.json, 那么: category = BlockstateResourceCategory.class, relativePath = mkz_bogey.json(除了第一个之后的 path)
ResourceCategory 通过 AllResourceCategory 进行注册为一个 Map<String, ResourceCategory>, 若文件夹类型不存在，则报错: Unable to found category for "name"
3. 然后使用 ResourceCategory::getResourceTargetPath(SourceResource) -> Path 返回的 Path 为 Key, Set<SourceResource> 为 Value, 进行输出文件归类

## 处理阶段
使用线程池，批量提交 3 中的 Set<SourceResource>, Path，然后:
1. 使用 ResourceCategory::getTransformers() 获得 TreeSet<Transformer>
然后根据优先级进行排序
从前往后对 Transformer 调用 Transformer::shouldTransform(Set<SourceResource>) -> boolean, 如果 其中一个为 true, 则将 SourceResource 加载到字节数组，并交给 Transformer 进行 Transformer::transform(Set<LoadedResource> loadedResources) -> Set<LoadedResource>, loadedResources 为 record(SourceResource source, byte[] data)
内置的 Transformer 有: JSONMergeTransformer, 将所有文件读入为 JSON 然后合并为一个，其中 shouldTransform = resources.length() > 1

2. 如果 resource.length() < 0, 跳过; 如果没有 Transform 被调用, 且 resource.length() == 1, 则使用 Files.copy() 复制到对应文件夹

如果 resource.length() == 1, 且有 transform 被调用，则保存 bytes[]

否则(resource.length() > 1) ，则报错: Unable to resolve the conflict. Source: [...files], Target: [file]
